### TypeScript-এ `interface` এবং `type` এর মধ্যে পার্থক্য

TypeScript-এ `interface` এবং `type` দুটোই টাইপ তৈরি করতে ব্যবহৃত হয়, কিন্তু এদের মধ্যে কিছু গুরুত্বপূর্ণ পার্থক্য আছে:

| বৈশিষ্ট্য                          | `interface`                                     | `type` (type alias)                   |
| ---------------------------------- | ----------------------------------------------- | ------------------------------------- | ------------------------------------------- |
| অবজেক্ট টাইপ ডিক্লেয়ার             | খুবই সাধারণ এবং পছন্দের উপায়                    | সম্ভব, কিন্তু কম ব্যবহৃত              |
| এক্সটেন্ড করা যায় (`extends`)      | হ্যাঁ, সহজেই (`interface A extends B`)          | হ্যাঁ, কিন্তু `&` (intersection) দিয়ে |
| ইমপ্লিমেন্ট করা যায় (`implements`) | ক্লাসে সহজেই ইমপ্লিমেন্ট করা যায়                | সম্ভব, কিন্তু কম সাধারণ               |
| একই নামে একাধিক ডিক্লেয়ার          | সম্ভব (অটোমেটিক মার্জ হয় – Declaration Merging) | অসম্ভব (ডুপ্লিকেট এরর দেয়)            |
| ইউনিয়ন টাইপ (`string               | number`)                                        | সম্ভব                                 | খুবই সহজ এবং এটার জন্যই সবচেয়ে বেশি ব্যবহৃত |
| টাপল, প্রিমিটিভ, কন্ডিশনাল টাইপ    | সম্ভব নয়                                        | সম্ভব                                 |
| পারফরম্যান্স                       | সাধারণত একটু দ্রুত                              | সামান্য ধীর হতে পারে (জটিল ক্ষেত্রে)  |
| অগমেন্টেশন (global augmentation)   | সহজেই সম্ভব (module augmentation)               | সম্ভব নয়                              |

### কখন কোনটা ব্যবহার করবেন?

| পরিস্থিতি                                       | প্রস্তাবিত                  |
| ----------------------------------------------- | --------------------------- |
| অবজেক্টের শেপ ডিফাইন করা (সাধারণ ক্ষেত্রে)      | `interface` (বেশি পঠনযোগ্য) |
| ক্লাসে ইমপ্লিমেন্ট করতে হলে                     | `interface`                 |
| একই নামে একাধিকবার ডিক্লেয়ার করতে হলে           | `interface` (মার্জ হবে)     |
| ইউনিয়ন, টাপল, প্রিমিটিভ বা কন্ডিশনাল টাইপ লাগলে | `type` (অবশ্যই)             |
| উপরের সবকিছু মিশিয়ে জটিল টাইপ বানাতে হলে        | `type`                      |

### সারাংশ (সহজ নিয়ম)

- **৯০% ক্ষেত্রে অবজেক্ট হলে → `interface` ব্যবহার করুন**
- **ইউনিয়ন, টাপল, বা জটিল টাইপ হলে → `type` ব্যবহার করুন**

```ts
// interface → ভালো
interface User {
  id: number;
  name: string;
}

// type → ভালো
type ID = string | number;
type Point = [number, number];
type Status = "loading" | "success" | "error";
```
